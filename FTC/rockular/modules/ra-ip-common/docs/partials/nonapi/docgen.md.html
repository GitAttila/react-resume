Generating documentation

To generate the documentation, run gulp buildDocs from command line. It is a wrapper task for running grunt-ngdocs plugin.

Output location

Currently, the grunt-ngdocs is configured to output the generated documentation to dist/docs folder. In order to view the documentation, the directory containing the docs has to be hosted on a web server.

Markdown (.md) to .ngdoc conversion

Documentation in this section is generated from markdown-formatted documents, which are bare .md files. grunt-ngdocs only recognizes .ngdoc files, so .md files need to be renamed and correctly annotated using ngdoc tags. This is done by gulp, but in order to have gulp correctly annotate a particular markdown file, you need to define the document's title. You do it by defining a meta tag with name="ngdocTitle" and content attribute being the title:

<meta name="ngdocTitle" content="Generating documentation" />

NOTE: you must use double quotes to enclose attribute values. Using apostrophes will cause gulp not to recognize the tag.

If you don't provide text in content attribute or forget to provide the tag at all or the tag is malformed and not recognized, the document will be skipped and will not be included in the output.
