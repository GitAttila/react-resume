raIpQuery
directive in module RA.IPDisplay

Description

The raIpQuery directive is responsible for rendering the Query Widget.

Note: By default the query widget header in design mode contains pure text boxes. When there is a possibility of usage of graphical model picker it might be used. All you need to do is specify the controller, template URL and window class in MODEL_PICKER constant in your application. These are then used in raOverlaySvc. This model picker dialog is used for both selecting a type and property. For example it might look like: 
        angular.module("myModule").constant(
             "MODEL_PICKER", {
                 controller: "modelPickerCtrl",
                 templateUrl: "path/to/the/template",
                 windowClass: "classOfMyWindow"
             }
        )

Input:

As a config parameter to the controller is passed object with 2 properties:
  • currentFqn: FQN of context in which the model picker should be opened
  • selection: FQN of selected item (if exists)
For the selection of type there is also passed property pickerMode with structure: 
            {
                itemsTab: {
                    selectable: true,
                    active: true
                }
            }

Expected output:

Object with 2 properties:
  • fqn: FQN of selected item or property
  • type: FQN of the type in type system for items or "string"/"number"/"boolean" for properties

Usage

as element:
<ra-ip-query
       rb-parameters="{object}"
       ri-fqn="{string}"
       rb-is-active="{boolean}"
       ro-schema="{function}"
       ro-examples="{function}"
       [rb-style="{string}"]>
</ra-ip-query>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
rbParametersobject

An object representing common parameters and the parameters of Query Widget.

Query Widget Specific Parameters

Property Type Details
boundType String Type of the item which should be queried (FQN in type system)
boundProperty String Relative FQN (to the found item) which should be used as a schemaType property
directive String Name of the Widget to be used to display Query results
schemaType String Type of the selected property ("number", "boolean", "string" or FQN of type from type system - based on this value the schema of selected widget is picked)
items Array Inner properties related to the specific widget selected in directive. The object in this array will be used for each item in query widget. Note that relative FQNs in this array are relative to the found item instance
common Array Common parameters which will be passed to inner widget

Note that caption and color properties are properties from "item" schema. These will be copied to each matched item from query result. On the other hand chartTitle and lastSeconds are properties from common schema and therefore these will be copied only to common section of inner widget.

riFqnstring

FQN the widget is relative to.

rbIsActiveboolean

Flag which shows whether the widget should be active.

roSchemafunction

Function which should be called to initialize the widget schema in outside environment.

roExamplesfunction

Function which should be called to initialize the widget examples in outside environment.

rbStyle
(optional)
string

String indicating which class should be used in widget.
