trendLogicSvc
service in module RA.IPDisplay

Description

Basically this service is the functional part of raIpTrend directive. Scope passed around is the scope which directive is creating. Besides the binded variables (like rbParameters, riFqn and so on) scope holds up the information which is passed into or related to Highcharts.Chart object:

  • scope.fqnsList is an array with fqn for every series. This list keeps the order of the fqns.
  • scope.fqns is a hash table, where keys are indices from fqnsList array, which holds information about series like unit, type and index (i.e. series order)
  • scope.series is an array of data suitable to push into Highcharts.Chart.series object, with some additional information (not defined in Series):
    • seriesType
    • seriesName
    • seriesUnit
    • stringData which can be reached in formatting functions so the entire process of creating such functions can be simplified.

  • prepareOptions(scope)

    Prepare options for trend, which will be easy to manipulate. fqns and fqnsList will be filled up regarding to the per fqn options and defaults params from rbParameters.

    In order to figure out the time frame for the Highcharts trend, there are three parameters are exposed to the user for change in rbParameters common section:

    • startTime - if defined it means start date of time period
    • endTime - if defined it means end date of time period
    • lastSeconds - defines number of seconds that it should be calculated
                for now as an anchor time.

    If all three fields are defined, the start and end times takes precedence.

    Parameters
    ParamTypeDetails
    scopeObject

    scope

  • processData(scope, data)

    Process incoming time series data. This function may affects also scope.fqns[...] object in case of missing series type

    NOTE: This function doesn't care if timeSeriesSvc return bad data, it assumes that either it got data in following structure: { startTime: {Number}, // timestamps in fact endTime: {Number}, messages: [ {Fqn1}: [ [x1, y1], [x2, y2], ... [xn, yn] ], ... {Fqnm}: [ [x1, y1], [x2, y2], ... [xm, ym] ] ] }

    or the error handling will catch such issue (in fact, this kind of checks should belong to the timeSeriesSvc service or some specialized VP service anyway)

    The only brief check which are performed are for existence of first elements on the object (so startTime, endTime and messages).

    Parameters
    ParamTypeDetails
    scopeObject

    scope

    dataObject

    incoming data

    Returns
    Array

    data suitable for highcharts
