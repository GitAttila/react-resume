raIpFileTrend
directive in module RA.IPDisplay

Description

This raIpFileTrend component is responsible for rendering File Trend Widget. This widget parses CSV file and renders chart from its data. There is a specific service to handle widget constants, schema and examples, see RA.IPDisplay.service:raIpFileTrendConstSvc.

Usage

as element:
<ra-ip-file-trend
       rb-parameters="{object}"
       ri-fqn="{string}"
       rb-is-active="{boolean}"
       ro-schema="{function}"
       ro-examples="{function}"
       rb-design-mode="{boolean}"
       rb-timezone="{string}">
</ra-ip-file-trend>

Parameters

ParamTypeDetails
rbParametersobject

An object representing the parameters of File Trend Widget.

Common Parameters

Property Type Details Default value
lookBackwardHours Number Number of Seconds the value are being displayed for. Special treatment for value 0 (see startTime and endTime parameters) and -1 (see lastSeconds parameter). 600 sec (5 minutes)
lastSeconds (optional) Number Mandatory and used in case when lookBackwardHours equals -1 (Last N seconds). Number of Seconds the value are being displayed for when Last N seconds is chosen.
startTime (optional) Date Start time date. Mandatory and used in case when lookBackwardHours equals 0 (Custom range).
endTime (optional) Date End time date. Mandatory and used in case when lookBackwardHours equals 0 (Custom range).
hideTimeControl (optional) Boolean Should widget time control be hidden False
hasFileHeader (optional) Boolean Flag indicating that file has a row which contains header True
fileHeaderRowNumber (optional - mandatory when hasFileHeader is true) Number File header row number - at which row there is file header. All lines before “fileHeaderRowNumber” are ignored by file parser. 1
timeBase (optional) String Definition where to take datetime values from. Possible values 'absoluteTime' (values are separete datetime definitions at each row/column) and 'timePlusIncrement' (there is only one datetime base and there is a column with increments) 'timePlusIncrement'
absoluteTimeColumn (mandatory when timeBase = 'absoluteTime') Number Column number for absolute datetime 1
timePlusIncrement (mandatory when timeBase = 'timePlusIncrement') String Possible values: 'absoluteTime' (there is timeseries VQT taken as base datetime), 'cell' (there is base time in CSV file) 'absoluteTime'
intervalColumnNumber (mandatory when timeBase = 'timePlusIncrement') Number Column number for column with increse interval values 1
intervalColumnUnit (mandatory when timeBase = 'timePlusIncrement') String Interval column values unit. Possible values: 'milliseconds', 'seconds', 'minutes', 'hours', 'days' 'seconds'
cellRowNumber (mandatory when timePlusIncrement is 'cell') Number Row number for row with base datetime (row identification) 1
cellColumnNumber (mandatory when timePlusIncrement is 'cell') Number Column number for column with base datetime (column identification) 1
highlightIntervalZeroValue (optional, used when timeBase = 'timePlusIncrement') Boolean Flag indicating that zero based interval value have to be highlighted true

Common parameters dependency:

-> hasFileHeader == true
         -> fileHeaderRowNumber
-> timeBase == 'absoluteTime'
         -> absoluteTimeColumn
-> timeBase == 'timePlusIncrement'
         -> timePlusIncrement == 'absoluteTime'
             -> intervalColumnNumber
         -> timePlusIncrement == 'cell'
             -> cellRowNumber
             -> cellColumnNumber
             -> intervalColumnNumber
         -> highlightIntervalZeroValue
