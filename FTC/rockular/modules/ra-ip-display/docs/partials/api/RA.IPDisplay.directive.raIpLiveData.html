raIpLiveData
directive in module RA.IPDisplay

Description

This directive is responsible for binding data/live data to directive scope based on widget schema, current parameters, current fqn context and configuration. This directive does not create new scope. GetOnce bound data are stored to object 'drawData' and live data are bound to object 'liveData' that will be created on scope. As this is only attribute directive without own scope, it shares scope where it is defined.

Usage

as attribute
<ANY ra-ip-live-data
     ra-ip-live-data-context-fqn="{string}"
     ra-ip-live-data-params="{String}"
     ra-ip-live-data-schema="{String}"
     ra-ip-live-data-is-get-once="{Boolean}"
     ra-ip-live-data-is-active="{Boolean}"
     ra-ip-live-data-time-series="{Boolean}"
     ra-ip-live-data-time-series-config="{String}"
     ra-ip-live-data-set-api-fn-name="{String}"
     ra-ip-live-data-reinit-on-common-params-change="{Boolean}">
   ...
</ANY>

Parameters

ParamTypeDetails
raIpLiveDataContextFqnstring

Name of scope property where directive can find fqn (fallback to scope.fqn)

raIpLiveDataParamsString

Name of scope property where directive can find fqn (fallback to scope.params)

raIpLiveDataSchemaString

Name of scope property where directive can find schema (fallback to scope.schema)

raIpLiveDataIsGetOnceBoolean

Flag indicating if live data request should be loaded only once. Defaults to false - live data are loaded repeatedly.

raIpLiveDataIsActiveBoolean

Flag indicating activity of live data, in case of false there will be no live data loaded. Defaults to false.

raIpLiveDataTimeSeriesBoolean

Flag indicating if there is timeseries mode for live data.

raIpLiveDataTimeSeriesConfigString

Function which returns configuration object for timeseries request, mandatory in case of timeseries mode. This object should contain direct request parameters for timeSeriesSvc and will be filled with fqns by current parameters and may contain one extra property called "autoReloadInterval" which will be used as interval parameter for timeseries repeated requests.

raIpLiveDataSetApiFnNameString

optional attribute with name of function defined on scope which should be called to provide raIpLiveData API with two methods: "reinitialize" - does reinitialization of whole live data "unsubscribe" - unsubscribe all registered live or timeseries repeated handlers

raIpLiveDataReinitOnCommonParamsChangeBoolean

flag indicating if live data should be reinitialized on common parameters change (defaults to false, only parameters non-common section is watched for change). This settings can be set only in initialization.
