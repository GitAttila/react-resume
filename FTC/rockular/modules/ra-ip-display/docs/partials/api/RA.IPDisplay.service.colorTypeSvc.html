colorTypeSvc
service in module RA.IPDisplay

Description

Service which supports operations with Color type. It also provides backwards compatibility with the versions of displays which still use "string" colors.

Color type has following format: 

{
     color: "string describing the color in any CSS-like format",
     type: "item from RA.IPDisplay module constant COLOR_TYPES",
     index: "index of color in the palette"
}

Methods

  • colorEquals(c1, c2)

    Returns true if specified color representations are equal.

    Parameters
    ParamTypeDetails
    c1StringObject

    First color (Color object or hexadecimal string)

    c2StringObject

    Second color (Color object or hexadecimal string)

    Returns
    Boolean

    Returns true if following conditions are met:

    • Both colors describes same color
    • Both colors are of the same type

    Example 1 - following colors are equal (String colors are considering as FIXED_COLOR)

        c1 = "#123ABC";
    c2 = {
         color: "#123ABC",
         type: COLOR_TYPES.FIXED_COLOR
    }

    Example 2 - following colors are not equal (they have different type)

        c1 = {
         color: "#123ABC",
         type: COLOR_TYPES.FIXED_COLOR
    };
    c2 = {
         color: "#123ABC",
         type: COLOR_TYPES.THEME_COLOR,
         index: 10
    }

  • getColorObject(color, type, palette)

    Builds color object based on input parameters.

    Parameters
    ParamTypeDetails
    colorString

    Hexadecimal color description. E.g. "#123ABC".

    typeString

    Item from COLOR_TYPES. Default is FIXED_COLOR

    paletteArray

    Array containing strings representation of colors. (without "#" prefix)

    Returns
    Object

    Color object with following properties:

    • color: String describing the color in hexadecimal notation
    • type: Item from COLOR_TYPES
    • index: This property is defined only if type is equal to THEME_COLOR. It contains index of specified color in specified palette. If type is equal to THEME_COLOR and color is not present in specified palette, it throws RAError.

    Example: 

     {
     color: "#123ABC"
     type: COLOR_TYPES.THEME_COLOR
     index: 10
 }

  • getColorString(color)

    Returns the lowercase string representing the input color.

    Parameters
    ParamTypeDetails
    colorObjectString

    Color representation

    Returns
    String
    • If user provides color object, it returns its "color" property.
    • If user provides string representation it just returns this unchanged string (it is because of backwards compatibility).
    • If the color parameter is something else than valid color object or string it throws RAError.

  • getColorType(color)

    Returns the type of input color from COLOR_TYPES enumeration.

    Parameters
    ParamTypeDetails
    colorObjectString

    Returns
    String
    • If user provides color object, it returns its type property.
    • If user provides String representation it it returns FIXED_COLOR (it is because of backwards compatibility).
