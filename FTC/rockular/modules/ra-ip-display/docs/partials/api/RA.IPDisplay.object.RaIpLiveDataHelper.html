RaIpLiveDataHelper
object in module RA.IPDisplay

Description

Instance representing the RaIpLiveDataHelper and its API. Created by calling new raIpLiveDataHelper, see raIpLiveDataHelper.

Usage

RaIpLiveDataHelper(schemaParam, valueItemsParam, contextFQNParam);

Parameters

ParamTypeDetails
schemaParamObject

whole widget schema

valueItemsParamArray

widget items parameters

contextFQNParamString

current context FQN value

Methods

  • cloneDataWithoutBinding(source)

    Clone inner array/object data from one object into another, creates structure without its binding.

    Parameters
    ParamTypeDetails
    sourceArrayObject

    source to clone data from

    Returns
    Array|Object

    cloned object without binded properties

    Example: 

    source: [{
     testA: 'testA',
     testB: {innerTestB: true},
     testC: ['a', 'b']
}]
    Result: 
    [{
     testB: {},
     testC: []
}]

  • createFqnListAndMapping(addAutobind)

    Function to create list of FQNs and their mapping by schema to allow to fill into proper place in value object. Also returns immediately resolved autobind values for binding to 'Name' property.

    Parameters
    ParamTypeDetails
    addAutobindBoolean

    process data as for getOnce request, it adds autobinded values

    Returns
    Object

    with structure: 

    {
      fqnList: fqnList,
      //array of FQNs for server request
      mapping: mapping,
      //mapping for each fqnList item paired by same index as in fqnList
      resolvedAutobindValues: resolvedAutobindValues,
      // array of immediatelly resolved values in proper structure as value object
      duplicateFqnList: duplicateFqnList
      // duplicateFqnList - array of duplicate fqns and its mapping with structure:
      [ 5 : [<mapping>, <mapping>], ...] where by index there is pairing to same fqn in fqnList
  }

  • fillValueToObject(object, mapping, value)

    Function to fill specific property of given object by given mapping.

    Parameters
    ParamTypeDetails
    objectobject

    object to fill properties to

    mappingarray

    mapping structure, based on this it fills data, each index means one level of object Examples: 

    Mapping:
[
     {type: 'object', key: 'firstLvl'},
     {type: 'object', key: 'test'},
     {type: 'value', key: 'valueProperty'}
]
Result:
{
     firstLvl: {
         test: {
             valueProperty: 'valueToFill'
         }
     }
}

    Mapping:
[
     {type: 'object', key: 'firstLvl'},
     {type: 'array', key: 'test'},
     {type: 'value', key: 'valueProperty'}
]
Result:
{
     firstLvl : {
         test : [ 'valueToFill' ]
     }
}

    Mapping:
[
    {type: 'object', key: 'firstLvl'},
    {type: 'array', key: 'test'},
    {type: 'object', key: 0},
    {type: 'value', key: 'valueProperty'}
]
Result:
{
     firstLvl : {
         test : [
             { valueProperty : 'valueToFill' }
         ]
     }
}

    Beware of setting wrong mapping as this one: 

    [
     {type: 'object', key: 'firstLvl'},
     {type: 'array', key: 'test'},
     {type: 'value', key: 'stringKey'}
     //string key ('stringKey') in array will result in wrong array
]
    valueMixed

    value which will be filled to last mapping path
