v2.0.0

Features

  • New widgets: Timeline and Doughnut

  • Time control header: Timeline, Doughnut and Trend widgets now optionally contains header for control the time ranges for visualized timeSeries data

  • new service widgetsConstantsSvc used to get widgets schemas, examples and register widget aliases to handle schema and examples

  • enhanced raIpLiveData directive to support timeSeries

  • new service utilitySvc with basic utility methods

  • gridstackSvc now supports both MFT and Kendo D&D

  • transformationSvc now supports hooks which can be registered outside of ra-ip-display module

  • B-108343: Refactored raIpQuery widget. It now contains the configuration header in design time which enables users to select type, related property and widget type. The schema of the widget is counted based on selected widget. The relative FQNs in the rbParameters.items array are now relative to the query result, the former version was relative to the context of the whole display. This should help to better usability of the widget.

  • B-153129: raIpTrend now uses new reusable chevron menu component, widget chevron menu is now shown if there is not enough space, otherwise bar with properties is displayed at top of the widget

  • B-155332: Enhanced test page to support manual tests described in JSON files

Breaking Changes

  • removed liveDataGroupCtrl controller, use raIpLiveData directive instead

  • removed raIpTimeSeriesData directive, use raIpLiveData directive instead with timeSeries settings

  • removed raIpPropagateSchema, it is not needed anymore since the schema can be now obtained synchronously without a need of widget compilation

  • widgets should be compiled as elements because of new addition of Timeline and Doughnut widgets which are angular components and allow element definition only, for example compilation see raIpCompositeCell line 35

  • when implementing a widget, widget should also define its own constant service with specific pattern ConstSvc and implement these methods: getSchema (returns widget schema), getExamples (returns widgets examples). See Api documentation and "Other docs" section.

Deprecated

  • directives API roExamples function and usage is deprecated, will be removed in future. To get widget examples use widgetsConstantsSvc getWidgetExamples method.

  • don't rely on directives API roSchema function, will be used in special cases and will be optional in future. To get widget schema use widgetsConstantsSvc getWidgetSchema method.

v1.0.0

The initial version of RA.IPDisplay module. This version contains widgets which schema is already defined to use validation functions. If you want to adopt this module and you are using Composer, make sure that your Property Editor supports validations.

Features

  • widgetsConfigurationSvc function init now accepts parameters. If they are specified, the widgetLibrary on server is not used at all.

  • ra-disable-translate-recursively and ra-disable-translate classes in raIpComposite. These should be used to disabled 3d translate on iOS devices

Breaking Changes

  • gridstackSvc: composerInit, microComposerInit and compositeInit were merged into more parametrized createGridstack function.

  • grdistackWrapper object (result of gridstackSvc.createGridstack function) property gridstackWrapper.gridstackWrapper renamed to grdistackWrapper.wrapperElement

  • From old application's composerSvc functions getSchemaTypePropertyName and loadWidgetsSchema were divided into the new schemaSvc
