_checkPaginationEnabled
_checkPaginationEnabled()

Evaluate whether the pagination controls should be displayed. If the scroll height of the tab list is wider than the size of the header container, then the pagination controls should be shown.

This is an expensive call that forces a layout reflow to compute box and scroll metrics and should be called sparingly.

Returns : void
_checkScrollingControls
_checkScrollingControls()

Evaluate whether the before and after controls should be enabled or disabled. If the header is at the beginning of the list (scroll distance is equal to 0) then disable the before button. If the header is at the end of the list (scroll distance is equal to the maximum distance we can scroll), then disable the after button.

This is an expensive call that forces a layout reflow to compute box and scroll metrics and should be called sparingly.

Returns : void

The vertical icon navbar component are designed to add vertical side navigation to a fullscreen app. To set up a navbar we use component <ra-ftc-vertical-icon-navbar> which acts as a structural container for our navbar items.

Configuration

Input bindings

Parameter navbarConfig (not required)

{
  "position": "left",
  "responsibleType": "scrollbar"
}
  • position - navbar position, accept params: 'right', 'left', default: 'left' 
    const position: string;
  • responsibleType - defines component behavior when container size is changed, accept params: 'scrollbar', 'context_menu', 'context_menu', default: 'scrollbar' 
    const responsibleType: string;

Parameter navbarList: default []: (required)

[
  {
    "id": "home",
    "icon": "ra-icon-home",
    "label": "Home link"
  }
]

  • id - navbar item id

    const id: string;

  • icon - name of icon definition in ra-icons component or default material icons

    const icon: string;

  • label - label for the icon navbar item

    const label: string;

    Parameter activeItem: (not required)

default:

{
  "id": "",
  "icon": "",
  "label": ""
}

Output bindings

Callback (activeItemChange): onClickItem(output)

output return selected item:

{
  "id": "home",
  "icon": "ra-icon-home",
  "label": "Home link"
}

Examples

<ra-ftc-vertical-icon-navbar
  [navbarConfig]="navbarConfig"
  [navbarList]="navbarList"
  [activeItem]="activeItem"
  (activeItemChange)="onClickItem($event)">
</ra-ftc-vertical-icon-navbar>
