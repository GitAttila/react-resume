getOnce
getOnce(request: TimeseriesRequest, timezone?: string)

Method to send timeseries request to server, but only once.

Parameters :
Name Type Optional Description
request TimeseriesRequest No

timeseries request to be sent
timezone string Yes

timezone to apply to timeseries response data, only moment timezones are supported If working with timezone in datetime string it is recommended to use moment.parseZone to preserve timezone information. Moment timezones that start with "Etc/GMT" use POSIX-style signs in the Zone names and the output abbreviations, even though this is the opposite of what many people expect. POSIX has positive signs west of Greenwich, but many people expect positive signs east of Greenwich. For example, TZ='Etc/GMT+4' uses the abbreviation "GMT+4" and corresponds to 4 hours behind UT (i.e. west of Greenwich) even though many people would expect it to mean 4 hours ahead of UT (i.e. east of Greenwich).
Returns : Observable<TimeseriesResponse>

(Optional) Set the timezone to apply to datetime strings for timeseries response data. Returned date strings are in maximum javascript resolution possible supported by using moment library (Javascript date - milliseconds). Resolution for time after milliseconds will be lost. If working with timezone in datetime string it is recommended to use moment.parseZone to preserve timezone information. Moment timezones that start with "Etc/GMT" use POSIX-style signs in the Zone names and the output abbreviations, even though this is the opposite of what many people expect. POSIX has positive signs west of Greenwich, but many people expect positive signs east of Greenwich. For example, TZ='Etc/GMT+4' uses the abbreviation "GMT+4" and corresponds to 4 hours behind UT (i.e. west of Greenwich) even though many people would expect it to mean 4 hours ahead of UT (i.e. east of Greenwich).
