raIpWizard
directive in module RA.IPComponents

Description

This component is responsible for rendering wizard. The component expects to receive {object} config.

Usage

as element:
<ra-ip-wizard
       config="{object}">
</ra-ip-wizard>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
configobject
  • sharedViewConfig {object} object to be shared across all controllers - used to pass data to and between controllers
  • initialViewIndex {integer} index at which will wizard open its view on first render, defaults to 0
  • showButtons {boolean} flag indicating if wizard should have action buttons at bottom (prev/next actions)
  • hideSteps {boolean} flag indicating if wizard should hide steps
  • previousStepsAutoSuccess {boolean} flag indicating if wizard should mark previous steps automatically as success (defaults to false)
  • stepsConfig {array} array with steps definition, see RA.IPComponents.component:raIpSteps
  • views {array} array with definition for each step - if no template and templateUrl is set then default template 'components/raIpWizard/raIpWizardDefault.tpl.html' is used
  • template {string} optional definition for inline template to be used for wizard content view
  • templateUrl {string} url for template to be used for wizard content view, template is loaded only from $templateCache ("components/raIpWizard/raIpWizard.tpl.html" cannot be used as templateUrl)
  • controller {string} controller name, controller will be created
  • controllerAs {string} optional - alternative name for controller for template (controller scope alias), if not set then default $ctrl is used

There is wizard api which is passed to controller created for actual step as 'wizardApi' injection.
