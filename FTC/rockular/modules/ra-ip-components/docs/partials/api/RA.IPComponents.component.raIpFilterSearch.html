raIpFilterSearch
directive in module RA.IPComponents

Description

The component filters data, it has multiple modes, and filter methods have to be provided by user. Filter has three modes. Basic without combining filters, combining filters using input element and custom combining.

Basic mode – fulltext search

Very basic fulltext search. No menu to choose filter options. No keywords can be used in input line. Callback function has to be provided. Argument to callback function ($ctrl.callback) example: 

queryData: [{
    name: 'all',
    query: "controller"
}]

String mode – keywords in input

After clicking on filter option, its name will be added to input line. User can write query after that string. Multiple filters can be added. E.g. “name:text category:basic name:field”. Filter results depend on filter implementation. Query can be erased by clicking cancel icon. Default is fulltext search without option choosen. Argument to callback function ($ctrl.callback) example: 

queryData: [{
    name: "name",
    query: "ellipse"
},{
    name: "category",
    query: "general"
}]

Advanced mode – custom implementation of graphic representation of active filters

After clicking on filter option, user can write query which belongs only this filter, after clicking on another filter option and writing another query, this query will be overwritten. After clicking on enter or checkmark, data will be sent to callback function which handles the search. Argument to callback function ($ctrl.callback) example: 

queryData: [{
    name: "name",
    query: "controller"
}]

Cancel callback

When user clicks on cancel button, callback is send in this format: 

queryData: [{
    cancel: true
}]

delay and minChars

When not specified delay will be set to 300ms and minChars to 1. Whispers will be activated when query >= minChars. Callback will be activated when query >= minChars and after delay.

Usage

as attribute
<ANY ra-ip-filter-search
     filters="{array}"
     do-confirm="{boolean}"
     delay="{number}"
     debounce="{number}"
     min-chars="{number}"
     mode="{string}"
     reset="{boolean}"
     whisper-last-query="{boolean}"
     buttons="{object}"
     default="{number}"
     callback="{function}"
     whispers="{function}"
     set-api="{function}">
   ...
</ANY>

Parameters

ParamTypeDetails
filtersarray

An array of objects representing filter and its configuration. When no filter is choosen, callback will be called with default filter 'all' (search in all properties).

Property Type Details Default value
icon (optional) String CSS class of the filter icon undefined
name String Unique name of filter, will be used as ID undefined
label (optional) String Filter label will be visible in filter options menu same as name
color (optional) String Background color of filter icon undefined
placeholder (optional) String Input placeholder undefined
doConfirmboolean

Does user has to confirm search query by clicking enter or check icon? //default: false

delaynumber

Time in ms after which callback function will be called, if query length is same or higher than minChars. //default: 300

debouncenumber

Input debounce in ms. //default: 1000

minCharsnumber

Minimal number of chars after which callback function will be called. //default: 1

modestring

('basic'/'string'/'advanced') Which mode to use. //required

resetboolean

Only in advanced mode. If chosen filter should reset to default after sending callback.

whisperLastQueryboolean

Only in string mode. Should apply whisper only to last keyword in input. //default: false

buttonsobject

Makes buttons visible/invisible. They are visible by default.

Property Type Details Default value
okButton Boolean Visible? true
cancelButton Boolean Visible? true
defaultnumber

Only for advanced mode. Index of default filter from filters array. //default: 0 (first element in array)

callbackfunction

Function which handles filter action. Query is provided as attribute. When no filter is chosen, it get filter with name 'all' //required

whispersfunction

Function which returns whisper for given search query. Query is provided as attribute. Only when combineFilters.

setApifunction

Function which sets api to control filter. Has only one method - reinit. It can be used to reinitialize filter in different mode.
